🧩 从源代码到汇编：一个迷你编译器的实现

词法分析 · 语法分析 · 抽象语法树 · MIPS 代码生成
Flex Bison C++17 MIPS CMake ✅ 支持 if / while / 函数调用

这是一个类 C 语言的迷你编译器，能够将简化版的 C 语言源程序编译为 MIPS 汇编代码， 并可在 SPIM 模拟器上运行。整个项目手工实现了编译前端的主要阶段：词法分析、语法分析、抽象语法树 (AST) 构建， 以及目标代码生成。本文档既是项目讲解，也同步梳理了编译原理中与之对应的核心知识。

1. 项目概览：能做什么？

✅ 支持的语言特性

  • 基本数据类型：int、void
  • 变量声明与初始化：int a = 10;
  • 算术运算：+、-、*、/、%
  • 位运算：&、|、^、~
  • 逻辑运算：&&、||、!
  • 关系比较：<、<=、>、>=、==、!=
  • 控制流：if、if-else、while
  • 函数定义与调用（支持最多 4 个参数）
  • 内置函数：println_int(expr) 用于输出

🎯 输入输出

  • 输入：一个符合上述语法的 .c 源文件
  • 输出：对应的 MIPS 汇编代码（.s 文件），可在 SPIM 中运行

3. 核心代码讲解（带编译原理知识）

3.1 词法分析 —— 把字符流变成 Token 流

lexer.l   使用 Flex 工具编写

📖 编译原理知识：词法分析器的任务是读入源程序的字符序列，将它们组合成有意义的词法单元 (Token)。 每个 Token 包含一个类型（如 INT、ID）和一个可选的属性值（如变量名、数字值）。 正规表达式和有限自动机是词法分析的理论基础。

🔧 本项目的实现：

"int"                       { return INT; }
"void"                      { return VOID; }
"return"                    { return RETURN; }
"if"                        { return IF; }
"else"                      { return ELSE; }
"while"                     { return WHILE; }

[a-zA-Z_][a-zA-Z0-9_]*      { yylval.str = strdup(yytext); return ID; }
[0-9]+                      { yylval.num = atoi(yytext); return NUMBER; }

"="                         { return ASSIGN; }
"+"                         { return PLUS; }
"&&"                        { return LAND; }
"||"                        { return LOR; }
/* ... 其他运算符 */

解释：关键字（如 int）被识别为独立的 Token 类型；标识符 ID 匹配字母/下划线开头的字母数字串， 并将字符串副本存入 yylval.str；数字 NUMBER 则转换为整数值存入 yylval.num。 空白字符被直接跳过，未知字符报错。这样，一段源代码就被切分成了一个有序的 Token 序列，供语法分析器使用。

3.2 语法分析 —— 从 Token 到语法树

grammar.y   使用 Bison 工具编写

📖 编译原理知识：语法分析根据语言的上下文无关文法，检查 Token 序列是否符合语法规则， 并构建出抽象语法树 (AST)。AST 是源代码的树状结构表示，去除了括号、分号等对语义无影响的细节， 是后续语义分析和代码生成的重要数据结构。Bison 使用 LALR(1) 分析法，是经典的移进-归约分析器。

🔧 本项目的实现：

function_def
    : type_specifier ID LPAREN parameter_list RPAREN compound_stmt
        { $$ = new ASTFunction($2, $1, $4, static_cast<ASTCompoundStmt*>($6)); }

if_stmt
    : IF LPAREN expression RPAREN statement %prec LOR
        { $$ = new ASTIfStmt($3, $5, nullptr); }
    | IF LPAREN expression RPAREN statement ELSE statement
        { $$ = new ASTIfStmt($3, $5, $7); }

while_stmt
    : WHILE LPAREN expression RPAREN statement
        { $$ = new ASTWhileStmt($3, $5); }

/* 表达式层次：严格遵循优先级与结合性 */
logical_or_expr
    : logical_and_expr
    | logical_or_expr LOR logical_and_expr  { $$ = new ASTBinaryOp(Op::LOR, $1, $3); }
/* ... 其他优先级层次 */

解释：每个文法规则右侧对应一个 AST 节点的构造。例如，function_def 规则在识别到一个函数定义时， 会创建一个 ASTFunction 对象，包含函数名、返回类型、参数列表和函数体（复合语句）。 表达式文法通过分层（logical_or_expr → logical_and_expr → bit_or_expr → …） 天然实现了运算符优先级（如 && 优先级高于 ||）和左结合性。这也是 Bison 中 %left 声明的作用。

3.3 抽象语法树 (AST) —— 程序的结构化表示

ast.h   C++ 结构体定义

📖 编译原理知识：AST 是编译器的核心数据结构，它把程序的语法结构用树形方式固定下来。 每个节点代表一个语法构造（如表达式、语句、声明）。AST 与具体语法树 (CST) 的区别在于，AST 省略了括号、分隔符等仅用于解析的辅助符号， 只保留对语义有用的信息，因此更便于后续处理。

🔧 本项目的实现：

// 表达式节点
struct ASTBinaryOp : ASTExpr {
    Op op;          // 操作类型（ADD, SUB, LT, LAND, ...）
    ASTExpr* left;  // 左操作数
    ASTExpr* right; // 右操作数
};

struct ASTCallExpr : ASTExpr {
    std::string funcName;
    std::vector<ASTExpr*>* args;  // 实参列表
};

// 语句节点
struct ASTIfStmt : ASTStmt {
    ASTExpr* cond;      // 条件表达式
    ASTStmt* thenStmt;  // then 分支
    ASTStmt* elseStmt;  // else 分支（可为空）
};

struct ASTWhileStmt : ASTStmt {
    ASTExpr* cond;
    ASTStmt* body;
};

// 声明节点
struct ASTVarDecl : ASTDecl {
    std::string name;
    Type type;          // INT 或 VOID
    ASTExpr* init;      // 初始化表达式（可为空）
};

// 顶层结构
struct ASTFunction : ASTNode {
    std::string name;
    Type retType;
    std::vector<ASTParam*>* params;
    ASTCompoundStmt* body;   // 函数体
};

解释：AST 的类层次清晰地区分了表达式 (ASTExpr)、语句 (ASTStmt) 和声明 (ASTDecl)。 每个节点只存储必要的语义信息。例如 ASTBinaryOp 存储操作符和左右子节点，而不存储括号或空格。 这种设计使得后续的代码生成器可以统一地遍历这棵树，不必关心语法细节。

3.4 代码生成 —— 从 AST 到 MIPS 指令

CodeGen.cpp   AST 遍历 + 指令发射

📖 编译原理知识：代码生成是编译器的最后一个阶段，它将 AST 转换为目标机器的汇编代码。 这个过程需要处理寄存器分配、栈帧管理、函数调用约定等底层细节。 本项目目标平台是 MIPS，一种经典的 RISC 架构，其指令集简洁规整，非常适合教学和理解编译器后端。

🔧 本项目的实现：

void CodeGen::generateExpr(ASTExpr* expr, const std::string& reg) {
    if (auto* lit = dynamic_cast<ASTIntLiteral*>(expr)) {
        os << "  li " << reg << ", " << lit->value << "\n";
    }
    else if (auto* var = dynamic_cast<ASTVarRef*>(expr)) {
        os << "  lw " << reg << ", " << varOffsets[var->name] << "($sp)\n";
    }
    else if (auto* bin = dynamic_cast<ASTBinaryOp*>(expr)) {
        generateExpr(bin->left, "$t0");   // 生成左操作数代码，结果放 $t0
        generateExpr(bin->right, "$t1");  // 生成右操作数代码，结果放 $t1
        switch (bin->op) {
            case Op::ADD: os << "  add " << reg << ", $t0, $t1\n"; break;
            case Op::SUB: os << "  sub " << reg << ", $t0, $t1\n"; break;
            case Op::MUL: os << "  mul " << reg << ", $t0, $t1\n"; break;
            case Op::DIV: os << "  div $t0, $t1\n  mflo " << reg << "\n"; break;
            case Op::LT:  os << "  slt " << reg << ", $t0, $t1\n"; break;
            // ... 其他操作
        }
    }
    else if (auto* call = dynamic_cast<ASTCallExpr*>(expr)) {
        if (call->funcName == "println_int") {
            generateExpr((*call->args)[0], "$a0");
            os << "  li $v0, 1\n  syscall\n";          // 打印整数
            os << "  la $a0, newline\n  li $v0, 4\n  syscall\n"; // 换行
        } else {
            // 参数传递（最多4个）与函数调用
            int argIdx = 0;
            for (auto* arg : *call->args) {
                generateExpr(arg, "$t0");
                if (argIdx < 4) os << "  move $a" << argIdx << ", $t0\n";
                argIdx++;
            }
            os << "  jal " << call->funcName << "\n";
            os << "  move " << reg << ", $v0\n";
        }
    }
}

解释：代码生成器采用递归遍历 AST 的方式。对于每个表达式节点，它递归生成子表达式的代码， 并将结果约定存放在指定寄存器中（如 $t0、$t1）。 二元运算先求左右子表达式的值，再发射相应的 MIPS 算术指令。 函数调用则按 MIPS 调用约定，将前 4 个参数放入 $a0–$a3，然后用 jal 跳转， 返回值从 $v0 中取回。这种“递归下降 + 寄存器约定”的方式，是教学编译器中非常典型且易于理解的做法。

此外，栈帧管理在 generateFunction 中实现：函数入口处分配栈空间，保存 $ra 和 $fp， 并映射局部变量到栈上的固定偏移。出口处恢复栈帧并返回。

4. 完整测试用例：输入 → 输出

📥 输入 (test.c)

int main() {
    int a = 10;
    int b = 20;
    int c = a + b * 2;
    println_int(c);

    if (c > 30) {
        println_int(100);
    } else {
        println_int(200);
    }

    int i = 0;
    while (i < 3) {
        println_int(i);
        i = i + 1;
    }
    return 0;
}

📤 输出 (MIPS 汇编片段)

.data
newline: .asciiz "\n"
.text
.globl main
main:
  addiu $sp, $sp, -136
  sw $ra, 132($sp)
  sw $fp, 128($sp)
  move $fp, $sp

  li $t0, 10
  sw $t0, 16($sp)       # a = 10
  li $t0, 20
  sw $t0, 20($sp)       # b = 20
  lw $t0, 20($sp)
  li $t1, 2
  mul $t1, $t0, $t1     # b * 2
  lw $t0, 16($sp)
  add $t0, $t0, $t1     # a + (b*2)
  sw $t0, 24($sp)       # c = ...

  lw $a0, 24($sp)
  li $v0, 1
  syscall               # println_int(c)
  la $a0, newline
  li $v0, 4
  syscall

  # if (c > 30) ...
  lw $t0, 24($sp)
  li $t1, 30
  slt $t0, $t1, $t0
  beq $t0, $zero, .L1
  li $a0, 100
  li $v0, 1
  syscall
  j .L2
.L1:
  li $a0, 200
  li $v0, 1
  syscall
.L2:
  # while 循环 ...
  ...

⬆ 左侧是一个包含了变量、算术、条件、循环、函数调用的测试程序；右侧是编译器生成的 MIPS 汇编代码， 可以看到每个 C 语句都忠实地转换为了对应的 MIPS 指令序列。

5. 项目中的编译原理知识点总结

  • 词法分析：正规表达式、有限自动机、Token 识别——由 Flex 工具实现。
  • 语法分析：上下文无关文法、LALR(1) 分析、移进-归约、抽象语法树——由 Bison 实现。
  • 语义分析（本实验简略）：类型检查、作用域管理、符号表——通过 C++ 数据结构模拟。
  • 中间代码与目标代码生成：AST 遍历、寄存器约定、栈帧管理、函数调用约定——手工实现 MIPS 代码生成。
  • 运算符优先级与结合性：通过文法分层（非终结符链）和 Bison 的 %left/%right 声明实现。
  • 控制流翻译：if 和 while 转换为条件分支 + 无条件跳转（beq/bne + j）。

可以说，这个项目覆盖了编译原理课程中“前端”和“后端”的主要环节，虽然做了很多简化（如不优化、不进行复杂的寄存器分配）， 但完整地呈现了一个真实编译器的工作流程，对于理解编译原理的实践意义非常有帮助。

6. 技术难点与收获

  • 难点1：运算符优先级的处理 —— 通过 Bison 的 %left/%right 和 %prec 精细控制， 同时在文法中设计了 8 个优先级层次（从 logical_or_expr 到 primary_expr），确保了所有运算符的正确结合与优先级。
  • 难点2：MIPS 栈帧的精确管理 —— 局部变量、临时寄存器、返回地址、帧指针都需要在栈上安排合理的偏移， 稍有不慎就会导致汇编运行出错。通过 varOffsets 映射表统一管理变量到栈偏移的映射，保证了正确性。
  • 难点3：函数调用与参数传递 —— 需要同时处理被调用函数（callee）的参数接收和调用者（caller）的参数传递， 严格遵循 MIPS 调用约定（$a0–$a3 传参，$v0 返回值）。

收获：通过亲手实现一个完整的编译器前端，对编译原理课上的“正规表达式”、“上下文无关文法”、“递归下降”、“三地址代码”等概念有了非常具象的认识。 更重要的是，理解了高级语言如何一步步被翻译成机器指令，这对后续学习计算机体系结构、操作系统甚至逆向工程都打下了坚实的基础。

8. 总结与延伸

这个迷你编译器项目，虽然规模不大，但完整覆盖了从源代码输入到汇编代码输出的全过程。 它用代码验证了编译原理课程中的核心理论，也暴露了真实编译器中需要权衡的许多工程细节。

如果继续扩展，可以进一步增加：

  • 更多数据类型（char、float、数组、指针）
